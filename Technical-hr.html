Technical Interview Questions: DSA, OOP, DBMS & Web

Grow your technical rounds for developer and testing roles. This part has 100 important questions with easy and clear answers on key Computer Science topics. Focus on understanding the main idea, not just memorizing answers.

Data Structures & Algorithms (DSA)

1. What is a 'Data Structure'?
DSA Fundamentals

A Data Structure is a special way of organizing and storing data in a computer so that it can be used efficiently. Examples are arrays, linked lists, and trees.

2. What is Time Complexity and how is it measured?
DSA Fundamentals

Time Complexity measures how fast an algorithm runs based on the size of the input data. We use **Big O Notation** (like $O(n)$ or $O(n^2)$) to describe this performance in a general way.

3. Explain the concept of a 'Linked List'.
DSA Structures

A Linked List is a collection of items, where each item (**node**) holds the data and a reference (**pointer**) to the next item. Unlike arrays, items are not stored next to each other in memory.

4. What is the main difference between a Stack and a Queue?
DSA Structures

A **Stack** is LIFO (**Last-In, First-Out**), like a stack of plates. The last item added is the first one removed. A **Queue** is FIFO (**First-In, First-Out**), like people waiting in a line.

5. What is a 'Binary Search Tree' (BST)?
DSA Trees

A BST is a special type of tree structure where for every node, all values in its **left** sub-tree are smaller than the node, and all values in its **right** sub-tree are larger. This makes searching very fast.

6. When would you use a Hashing (Hash Map/Dictionary)?
DSA Hashing

I would use a Hash Map when I need **very fast lookups** (searching, inserting, or deleting data) based on a unique key. It provides near $O(1)$ constant time complexity on average.

7. Explain Merge Sort briefly.
DSA Algorithms

Merge Sort is a 'Divide and Conquer' algorithm. It keeps dividing the list into smaller halves until each piece has only one element. Then, it repeatedly **merges** the sorted pieces back together to get the final sorted list.

Object-Oriented Programming (OOP)

8. What is the fundamental concept of OOP?
OOP Fundamentals

OOP is a programming method that uses **objects** (which are instances of classes) to organize code. It focuses on data and the functions that work on that data, rather than logic and actions alone.

9. What is a 'Class' vs. an 'Object'?
OOP Fundamentals

A **Class** is the blueprint or template (e.g., Car model design). An **Object** is a real-world instance built from that template (e.g., your actual Maruti car). You can create many objects from one class.

10. Define 'Polymorphism' with a simple example.
OOP Pillars

**Polymorphism** means 'many forms'. Example: The `draw()` method can behave differently when called on a Circle object than when called on a Square object, even though the method name is the same.

11. What is 'Inheritance' and why is it used?
OOP Pillars

Inheritance is a way for a new class (**Child Class**) to take properties and methods from an existing class (**Parent Class**). It helps us **reuse code** and maintain a clear relationship between different types of objects.

12. What is 'Encapsulation'?
OOP Pillars

Encapsulation means **bundling the data and the methods** that work on that data into one unit (the class). It also means restricting direct access to the data from outside, protecting it from accidental changes.

13. What is an 'Abstract Class'?
OOP Concepts

An Abstract Class is a class that **cannot be used directly** to create objects. It is only used to be inherited by other classes. It often contains abstract methods that the child classes *must* implement.

Database Management Systems (DBMS & SQL)

14. What is a 'Primary Key'?
DBMS/SQL Fundamentals

A Primary Key is a column (or set of columns) that **uniquely identifies** every record (row) in a database table. It cannot contain null (empty) values and must be unique.

15. What is the difference between DROP, DELETE, and TRUNCATE commands?
SQL Commands

**DELETE** removes rows and can be rolled back. **TRUNCATE** removes *all* rows fast and cannot be rolled back easily. **DROP** removes the entire table structure from the database.

16. Explain 'INNER JOIN' in SQL.
SQL Joins

An INNER JOIN returns only the rows that have **matching values** in both of the tables being joined. It's the most common type of join used for combining related data.

17. What are the ACID properties in transactions?
DBMS Transactions

ACID stands for **Atomicity** (all or nothing), **Consistency** (only valid data is saved), **Isolation** (transactions don't interfere with each other), and **Durability** (changes are permanent). These ensure reliable data processing.

18. What is 'Normalization'?
DBMS Design

Normalization is a database design technique used to **organize tables efficiently** and reduce data redundancy (avoid repeating data) and improve data integrity.

19. What is a 'Foreign Key'?
DBMS/SQL Fundamentals

A Foreign Key is a column that links two tables together. It points to the **Primary Key** of another table, maintaining the relationship and consistency between the data.

Web/Networking/OS Fundamentals

20. What is the difference between HTTP and HTTPS?
Networking/Web

HTTP (**HyperText Transfer Protocol**) is the basic protocol for data transfer on the web. **HTTPS** is the secure version; the 'S' stands for **Secure**, meaning the data is encrypted using SSL/TLS before sending.

21. What is the purpose of a 'Session' and a 'Cookie' in a web app?
Web Development

Both are used to store user information, as HTTP is stateless. **Cookies** are small data pieces stored on the user's browser. **Sessions** store data on the server and use a cookie as a unique key to identify the user.

22. What is a 'Process' and a 'Thread'?
OS Concepts

A **Process** is an instance of a computer program being executed (like running Chrome). A **Thread** is a smaller part of a process that can run independently. Multiple threads can exist within one process, sharing resources.

23. What is 'Deadlock' in an Operating System?
OS Concepts

A Deadlock happens when two or more processes are waiting for each other to release a resource, and none of them can proceed. It's like two cars blocking each other in a narrow lane.

24. Differentiate between Frontend and Backend.
Web Development

**Frontend** is what the user sees and interacts with (UI, built with HTML, CSS, JS). **Backend** is the server, application logic, and database where data is stored and processed (Python, Java, Node.js).

25. What is an IP address?
Networking

An IP (**Internet Protocol**) address is a unique numerical label assigned to every device (like your phone or laptop) connected to a computer network, used for identification and location addressing.

26. What are the common HTTP status codes (200, 404, 500)?
Networking/Web

**200 OK** means the request succeeded. **404 Not Found** means the requested resource doesn't exist. **500 Internal Server Error** means the server failed to fulfill the request due to an unexpected condition.

Simulated Remaining Technical Questions (Q27 - Q100)

27. What is a 'Wrapper Class' in Java?
Java/OOP

A Wrapper Class converts Java's primitive data types (like int, char) into objects. This is needed when working with collections that only store objects.

28. When is an algorithm considered 'stable' in sorting?
DSA Algorithms

A sorting algorithm is stable if it keeps the relative order of records with equal keys (values) the same after sorting.

29. What is the key use of 'Abstraction' in OOP?
OOP Pillars

Abstraction means showing only essential information to the user and hiding the complex background details. It simplifies using the code.

30. What is 'Garbage Collection'?
Java/Memory Mgmt

Garbage Collection is an automatic memory management process in languages like Java that automatically reclaims memory occupied by objects that are no longer being used by the program.

31. What is the time complexity of Quick Sort?
DSA Complexity

In the average and best case, Quick Sort is $O(n \log n)$. In the worst case, it can be $O(n^2)$.

32. Name two different ways to implement a 'Graph'.
DSA Structures

Graphs can be implemented using an **Adjacency Matrix** (good for dense graphs) or an **Adjacency List** (better for sparse graphs).

33. What is method 'Overloading' vs 'Overriding'?
OOP Polymorphism

Overloading: Same method name, but different parameters (in the same class). Overriding: Same method name and same parameters (in parent and child classes).

34. What is a 'constructor'?
OOP Concepts

A constructor is a special method in a class that is called automatically when an object is created. Its main purpose is to initialize the object's variables.

35. What is the key advantage of a 'Dynamic Array' (like ArrayList)?
DSA Structures

The main advantage is that its size can grow or shrink automatically during runtime, unlike traditional static arrays.

36. What is the difference between a 'WHERE' clause and a 'HAVING' clause?
SQL Filtering

The **WHERE** clause is used to filter records *before* grouping. The **HAVING** clause is used to filter groups *after* grouping has been done (often used with aggregate functions).

37. What is 'Indexing' in a database?
DBMS Concepts

Indexing is like creating a book's index. It speeds up data retrieval operations by quickly pointing to where the data is stored in the table, but it slows down data modification (INSERT/UPDATE).

38. Explain 'Left Join'.
SQL Joins

A LEFT JOIN returns all rows from the **left** table, and the matching rows from the right table. If there is no match, the right side returns NULL.

39. What is a 'View' in SQL?
SQL Objects

A View is a **virtual table** whose contents are defined by an SQL query. It doesn't store data itself; it just shows the data from real tables based on the query definition.

40. What is 'Stored Procedure'?
SQL Programming

A Stored Procedure is a prepared SQL code block that you save in the database. You can run it multiple times whenever needed, which improves performance and security.

41. What is the main purpose of 3rd Normal Form (3NF)?
DBMS Normalization

The purpose of 3NF is to eliminate fields that are not fully dependent on the primary key, thereby reducing data redundancy.

42. What is the 'DOM' in web development?
Web Development

The **Document Object Model (DOM)** is a programming interface for HTML and XML documents. It represents the page so that programs (like JavaScript) can change the structure, style, and content of the document.

43. What is the 'OSI Model' and its key layers?
Networking

The **Open Systems Interconnection (OSI)** model describes how information moves over a network. The key layers are Application, Transport, Network, and Data Link (among others).

44. What is 'Multithreading' and why is it useful?
OS Concepts

Multithreading allows a single program to have multiple parts running at the same time. It's useful for improving the performance and responsiveness of applications by utilizing CPU cores effectively.

45. What is 'REST API'?
Web Services

REST (**Representational State Transfer**) is an architectural style for designing networked applications. It often uses standard HTTP methods (GET, POST, PUT, DELETE) to communicate between systems.

46. How does 'CSS Box Model' work?
Frontend Basics

The Box Model describes how HTML elements are laid out on a page. Every element is wrapped by Content, Padding, Border, and Margin (from inside to outside).

47. What is 'Recursion'?
Programming Concepts

Recursion is a method where the solution to a problem depends on solutions to smaller instances of the same problem. In coding, it means a function calling itself.

48. What is the use of 'Final' keyword in Java?
Java/OOP

The 'final' keyword can be used to restrict the user. If used with a variable, the value cannot be changed. If used with a class, it cannot be inherited. If used with a method, it cannot be overridden.

49. Define 'Encapsulation' in Python/JavaScript terms.
OOP Concepts

It means using conventions (like a single underscore `_` or double underscore `__` in Python) to tell other developers that a variable is intended to be private or internal to the class.

50. What is a 'Primary Partition' in an OS?
OS Concepts

A Primary Partition is a physical section of a hard disk that can be used as a separate operating system drive. It's where the OS is typically installed and boots from.

51. What is a 'Cookie'?
Web Basics

A small file or piece of data stored on the user's computer by a web browser, used to remember user information like login status or preferences.

52. Why is $O(\log n)$ considered very efficient?
DSA Complexity

It means the time taken to run the algorithm grows very slowly as the input size increases (e.g., in Binary Search, doubling the input only adds one step).

53. What is 'Authentication' vs 'Authorization'?
Security/Web

**Authentication** verifies who you are (login/password). **Authorization** determines what you are allowed to do (permissions/access rights) after you are logged in.

54. Name four main features of OOP.
OOP Pillars

The four main pillars are: Encapsulation, Inheritance, Polymorphism, and Abstraction.

55. Write an SQL query to get all unique cities from a 'Customers' table.
SQL Queries

SELECT DISTINCT City FROM Customers;

56. What is 'Cascading Style Sheets' (CSS)?
Frontend Basics

CSS is used to describe how HTML elements should be displayed on screen, paper, or in other media (i.e., it handles the look and feel of a webpage).

57. What is 'Denormalization' and when is it used?
DBMS Design

It is the process of intentionally adding redundancy to a normalized database, often to improve **read performance** and speed up complex queries (at the cost of some update speed).

58. What is the 'Virtual Machine' in Java (JVM)?
Java Concepts

The JVM is an abstract machine that allows Java code (bytecode) to be run on any platform (Windows, Linux, Mac). This enables Java's 'Write Once, Run Anywhere' capability.

59. How does an 'Index' improve query speed?
DBMS Indexing

It allows the database to look up rows directly using the index value, instead of scanning the entire table from top to bottom (full table scan).

60. What is 'Recursion' and when can it cause an error?
Programming Concepts

Recursion is when a function calls itself. It causes a **Stack Overflow Error** if the base condition (the stopping point) is never reached, leading to infinite calls.

61. What is the function of the 'Operating System' (OS)?
OS Concepts

The OS manages all hardware and software resources of the computer. It handles memory, controls input/output devices, and manages running processes.

62. What is the use of 'Static' keyword in Java/C++?
Language Basics

The 'static' keyword means that the member (variable or method) belongs to the **class itself**, not to a specific object instance. It is shared across all objects of that class.

63. Explain 'TCP' vs 'UDP' in networking.
Networking

**TCP** (Transmission Control Protocol) is reliable, ordered, and connection-based. **UDP** (User Datagram Protocol) is faster, unreliable, and connection-less (used for streaming).

64. What is a 'Trigger' in SQL?
SQL Programming

A Trigger is a special type of stored procedure that automatically runs when an event occurs in the database (like an INSERT, UPDATE, or DELETE operation).

65. Give an example of a $O(n)$ algorithm.
DSA Complexity

Searching for an element in an unsorted array (Linear Search) is $O(n)$, because in the worst case, you might have to check every single element once.

66. How do you implement a Queue using two Stacks?
DSA Structures

Use one stack for enqueue (pushing) and a second stack for dequeue (popping). Before dequeuing, move all elements from the first stack to the second stack to reverse the order.

67. What are the four types of SQL Joins?
SQL Joins

INNER JOIN, LEFT (OUTER) JOIN, RIGHT (OUTER) JOIN, and FULL (OUTER) JOIN.

68. What are 'SOPs' (Standard Operating Procedures)?
IT Process

SOPs are detailed, written instructions that guide employees on how to perform a routine activity, ensuring consistency and compliance in business processes.

69. What is the key principle of 'Agile Methodology'?
Software Methodologies

The key principle is delivering working software frequently and adapting to changing requirements through continuous feedback and collaboration.

70. Why is 'HTML' not considered a programming language?
Web Basics

HTML is a **markup language**. It defines the structure and content of a webpage. It doesn't perform calculations or execute logic; it just formats data.

71. What is the difference between Array and ArrayList (Java)?
Java/DSA

Array is fixed-size and stores primitives/objects. **ArrayList** is dynamic (resizable) and only stores objects (using a dynamic array structure internally).

72. What are 'Semaphores'?
OS Concepts

Semaphores are synchronization tools used to control access to shared resources by multiple processes in a concurrent environment, preventing conflicts.

73. Explain 'Session Management' in a web application.
Web Security

It's the process of tracking a user's activity and status across multiple HTTP requests, typically by generating a unique Session ID after login and storing it in a cookie.

74. What is the purpose of the 'GROUP BY' clause in SQL?
SQL Queries

It groups rows that have the same values into summary rows, often used with aggregate functions like COUNT, MAX, MIN, SUM, or AVG.

75. What is the time complexity of searching in a BST?
DSA Complexity

In the average case, it is $O(\log n)$. In the worst case (if the tree is highly unbalanced), it degrades to $O(n)$.

76. What is 'Interface' in OOP?
OOP Concepts

An Interface is a completely abstract class that is used to define a contract. It declares a set of methods that any class implementing the interface must implement.

77. What is 'DNS'?
Networking

**Domain Name System** (DNS) translates human-readable domain names (like google.com) into machine-readable IP addresses.

78. Explain the 'Two-Phase Commit' protocol.
DBMS Transactions

A protocol that ensures all parties in a distributed transaction either **commit** (complete) the change or **abort** (roll back) it entirely, maintaining atomicity.

79. What is a 'Class Method' vs an 'Instance Method' (Python)?
Python/OOP

A Class Method works on the class itself (uses `cls`). An Instance Method works on a specific object (uses `self`).

80. Why is Binary Search faster than Linear Search?
DSA Algorithms

Binary Search eliminates half of the remaining data in each step ($O(\log n)$), while Linear Search checks one element at a time ($O(n)$).

81. What is the function of the 'Kernel' in an OS?
OS Concepts

The Kernel is the core part of the OS. It manages the communication between hardware and software, handling memory, CPU scheduling, and I/O operations.

82. What is 'SQL Injection'?
Security/DBMS

A security vulnerability where an attacker inserts malicious SQL code into a web form input field to bypass security or damage the database.

83. What are 'Unit Tests' and why are they important?
Software Testing

Unit Tests check if the smallest parts of code (functions or methods) work correctly and independently. They are essential for finding bugs early and making code safe for refactoring.

84. What is 'Client-Side' vs 'Server-Side' rendering?
Web Development

**Client-Side** rendering means the browser loads the blank page and uses JavaScript to build the content. **Server-Side** rendering means the server builds the full HTML page before sending it to the browser.

85. What is a 'Hash Collision' and how is it resolved?
DSA Hashing

A collision occurs when two different keys map to the same position in the hash table. It is resolved using techniques like **Chaining** (using a linked list at that position) or **Open Addressing** (finding the next empty slot).

86. What is the 'this' keyword in OOP (Java/JS)?
OOP Concepts

The 'this' keyword refers to the **current object instance**. It is used inside a method or constructor to refer to the object that is executing the code.

87. Explain 'Authentication' vs 'Authorization'.
Security/Web

**Authentication** proves who you are (login). **Authorization** defines what actions you are allowed to perform (access rights).

88. What is the difference between a 'Hub' and a 'Switch'?
Networking Devices

A **Hub** sends data to all connected devices (less efficient). A **Switch** only sends data to the specific device it is meant for (smarter and faster).

89. What is 'Method Signature'?
OOP Concepts

A method's signature is its name and the list of its parameters (number, type, and order). The return type is generally not considered part of the method signature.

90. Why is $O(n!)$ considered the worst time complexity?
DSA Complexity

The execution time grows extremely rapidly (factorially) with the input size. Algorithms with this complexity are only practical for very small input sets.

91. What is 'Virtual Memory' in an OS?
OS Concepts

Virtual Memory is a technique that allows a computer to compensate for physical memory shortages by temporarily transferring data from the RAM to disk storage (hard drive).

92. Explain 'Cross-Site Scripting (XSS)' attack.
Web Security

An XSS attack occurs when malicious script code is injected into a trusted website. This script runs in the user's browser, potentially stealing cookies or session information.

93. How is a 'Stack Overflow' error caused?
Programming Errors

It usually happens during **infinite recursion**, where a function keeps calling itself without a base (stopping) condition, eventually running out of the memory space (stack) allocated for functions.

94. What is 'Referential Integrity'?
DBMS Integrity

A concept that ensures relationships between tables remain consistent. It requires that a foreign key value in one table must either match a primary key value in the related table or be NULL.

95. What is a 'Load Balancer' in a web system?
Web/Architecture

A device or service that efficiently distributes incoming network traffic across multiple servers. This ensures no single server is overloaded, improving reliability and performance.

96. What is the 'Heap' data structure?
DSA Structures

A Heap is a complete binary tree where the parent node is either always greater than (Max Heap) or always smaller than (Min Heap) its children. It's often used to implement Priority Queues.

97. What is the primary use of 'Interface' in Java?
Java/OOP

It's used to achieve **multiple inheritance** (since Java doesn't allow multiple class inheritance) and define a standard set of behaviors (contracts) that a class must follow.

98. What is the difference between 'git pull' and 'git fetch'?
Version Control (Git)

**Fetch** downloads changes from the remote repository but doesn't integrate them into your local branch. **Pull** does both: it fetches the changes and then merges/integrates them automatically.

99. What are 'Threads' primarily used for in modern applications?
OS Concepts

Threads are used to handle multiple tasks concurrently within a single program (e.g., in a web browser, one thread loads images while another handles user input).

100. Write a basic SQL query to create an index on the 'EmployeeID' column of the 'Employees' table.
SQL Queries

CREATE INDEX idx_employeeid ON Employees (EmployeeID);
